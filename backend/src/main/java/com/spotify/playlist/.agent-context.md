# Playlist Service - Local Domain Context

## Core Responsibility
Playlist CRUD operations and track ordering within playlists.

## Domain Schema

| Entity | Fields | Rules |
| :--- | :--- | :--- |
| **Playlist** | `id`, `name`, `ownerId`, `isPublic`, `collaboratorIds` | Only owner can edit settings/delete. |
| **PlaylistTrack** | `playlistId`, `trackId`, `lexoRank`, `addedAt`, `updatedAt` | Ordering via LexoRank. |

## Value Objects
- `LexoRank`: Lexicographical rank string, max 128 chars (precision).

## Outgoing Domain Events
- `TrackAddedToPlaylist`: Fired on item addition.
- `TrackMovedInPlaylist`: Fired on rank update.

## Service boundaries
- Aggregates: `Playlist` (Aggregate Root), `PlaylistTrack`.
- Cross-service impact: Referenced by `track-service` (for tracks) and `user-service` (for owners).
