# Auth Service - Local Domain Context

## Core Responsibility
Authentication, Authorization, Identity Management, and Profile data.

## Domain Schema

| Entity | Fields | Rules |
| :--- | :--- | :--- |
| **User** | `id`, `email`, `password`, `displayName`, `avatarUrl`, `createdAt`, `updatedAt` | `email` unique/valid. `password` letter + number, min 8 chars. |

## Value Objects
- `Email`: RFC compliant, stored lowercase.
- `Password`: BCrypt hashed in infrastructure.

## Outgoing Domain Events
- `UserRegistered`: Published after successful sign-up.

## Service boundaries
- Aggregates: `User`.
- Cross-service impact: Owns the `userId` used by all other services.
